# CLAUDE.md

This file gives coding agents project-specific guidance for this repository.

## Project overview

- Package: `polymarket-sdk-ts`
- Goal: publish a TypeScript SDK for Polymarket APIs
- Covered modules: `clob`, `gamma`, `data`, `bridge`
- Source of truth for API surfaces: Polymarket API Reference OpenAPI specs

## Repository structure

- `generate-sdks.ts`: fetches OpenAPI docs and generates SDK code
- `src/sdk/core.ts`: shared HTTP client and error handling
- `src/sdk/clob/generated/*`: generated CLOB client and types
- `src/sdk/gamma/generated/*`: generated Gamma client and types
- `src/sdk/data/generated/*`: generated Data client and types
- `src/sdk/bridge/generated/*`: generated Bridge client and types
- `src/sdk/index.ts`: public exports and unified `PolymarketSDK`
- `dist/*`: publish artifacts (generated by build)

## Commands

- Install deps: `pnpm install`
- Regenerate API modules: `pnpm run generate:sdk`
- Type check: `pnpm run check`
- Build package: `pnpm run build`
- Validate package content: `npm pack --dry-run`

## Development rules

- Treat generated files under `src/sdk/*/generated` as build output.
- Prefer changing `generate-sdks.ts` over manually editing generated clients/types.
- Keep public exports in `src/sdk/index.ts` stable and explicit.
- Keep package output compatible with both ESM and CJS (`dist/index.js`, `dist/index.cjs`).

## Release checklist

1. `pnpm run generate:sdk`
2. `pnpm run check`
3. `pnpm run build`
4. `npm pack --dry-run`
5. Bump version in `package.json`
6. `pnpm publish --access public`
